# Contributing

## Branching
- `main` stays clean and runnable
- Work in feature branches: `feat/<topic>`

## Commit rules
- No secrets (see 02-security.md)
- Keep examples in `/samples` only
- Any flow change must update:
  - `docs/03-flow-inventory.md`
  - `docs/05-change-log.md`

## Naming
Flows use prefix: `UIR - <purpose>`
