# Contributing

## Branching
- `main` is always runnable/documented
- Work in feature branches: `feat/<topic>`

## Commit rules
- No secrets (see SECURITY.md)
- Keep examples in `/samples` only
- Docs updates required for any flow change (docs/03-flow-inventory.md + docs/05-change-log.md)

## Naming
Flows use prefix: `UIR - <purpose>`
